You'll most probably be calling this more than anything else in this API. In response to this Smarkets will send you one of order-accepted, order-rejected, or order-invalid.

To prevent your application from accidently creating more orders than it really wanted to you should always locally associate your orders within your application with the order creation messages and outgoing seq of the sequenced payload you send to Smarkets for the current session.

If a sequenced payload with an order-create message has a different outgoing seq number then it will result in a new order being created.
